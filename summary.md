# AraNES — ядро, которое ускоряет LMS

AraNES — это модульное FastAPI‑ядро для образовательных платформ, где скорость развития важнее монолитной «традиции». Здесь плагины, единый контракт прав доступа и быстрые in‑process вызовы дают архитектуру, в которую легко влюбиться: гибкую, понятную и масштабируемую.

## Почему это важно
Большинство LMS построены как тяжелые монолиты. Каждая новая интеграция — боль, каждая фича — компромисс. AraNES решает эту проблему за счет архитектуры:

- Плагины подключаются и управляются как сервисы
- Права доступа расширяются динамически через общий реестр
- Внутренние вызовы идут без сетевого оверхеда

## Что AraNES дает продукту
- Быстрый запуск и понятный API
- Гибкое развитие без переписывания ядра
- Контроль над архитектурой, а не борьба с ней
- Нормальный путь к масштабированию функционала

## Как это работает
- Плагины грузятся из `services/` и включаются без перезапуска
- Permissions Registry собирает общую схему прав из моделей плагинов
- In‑process HTTP сохраняет FastAPI‑валидацию и убирает сеть
- JWT‑доступ разделяет открытые и защищенные endpoints

## Качественное сравнение с типичной LMS

| Критерий | Типичная LMS | AraNES |
|---|---|---|
| Расширяемость | Тяжелая, медленная | Плагины как сервисы |
| Изменения без рестарта | Обычно нет | Да |
| Внутренние вызовы | HTTP/сеть | In‑process |
| Права доступа | Жестко зашиты | Динамические |
| Архитектурная гибкость | Ограничена наследием | API‑first |

## Кому подходит
- Командам, которые строят LMS как платформу, а не как «сайт с курсами»
- Продуктам, где нужна быстрая интеграция новых модулей
- Стартапам, которые не хотят терять скорость из‑за монолита
